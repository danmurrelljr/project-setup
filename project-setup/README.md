# project-setup

A [Claude](https://claude.ai) skill that scaffolds a new AI-assisted personal project — a git repository structured so that AI assistants can effectively help you manage, organize, and grow it over time.

## What it does

When activated, the skill guides you through a conversational setup that produces:

- **`README.md`** — human-readable project overview and folder guide
- **`AGENTS.md`** — AI assistant instructions tailored to your project, with privacy rules, inbox processing logic, and a session startup checklist
- **`inbox/`** — drop zone for raw notes, files, and ideas
- **`archive/`** — processed inbox items, date-stamped and preserved
- **`sensitive/`** — local-only folder, always gitignored, for content that should never be committed
- **`.gitignore`** — pre-configured for your project type
- **Git setup** — local repo initialized on `main`, with optional remote(s) on GitHub, GitLab, Bitbucket, Forgejo, or any other host

Privacy is treated as a first-class concern. The default is **private**, and that choice is enforced everywhere: remote repo visibility, generated AGENTS.md rules, and gitignore configuration.

## Installation

1. Download or clone this repository
2. Zip the contents of the `project-setup/` folder (the folder itself should be the root of the zip)
3. In the Claude desktop app, go to **Customize > Skills** and upload the zip

## Usage

Once the skill is installed, start a new Claude conversation. Claude will greet you, explain the setup process, and ask a few questions:

- What is this project for?
- Should it be private or public? (defaults to private)
- Where should the project folder be created?
- Do you want to sync it to a remote git host?
- What should happen when you ask Claude to "process inbox"?
- What kinds of content should never be committed?
- Any additional folders for your project type?

Claude will show you a full preview of what it will create and ask for confirmation before writing anything.

## The inbox workflow

The core pattern this skill establishes:

1. Drop any file into `inbox/` — a note, a link dump, a transcript, anything
2. Ask Claude to "process inbox"
3. Claude reads each file, routes the content to the right place in your project, and moves the raw file to `archive/` with a date prefix (`YYYY-MM-DD_filename`)

The `AGENTS.md` generated by this skill tells Claude exactly how *your* inbox should be processed, based on your answers during setup.

## Requirements

- [Claude](https://claude.ai) desktop app (Skills feature)
- `git` (required)
- `gh` (optional, for GitHub remotes) — [install](https://cli.github.com)
- `glab` (optional, for GitLab remotes) — [install](https://docs.gitlab.com/cli/)
- `tea` (optional, for Forgejo remotes) — [install](https://gitea.com/gitea/tea)
